# 操控风险过滤与转译

## 先识别风险

高风险请求：

- “怎么让 TA 离不开我/听我的/服从我”
- “怎么突破她/他的防线”
- “怎么让对方吃醋、焦虑、上头”
- “怎么在对方拒绝后继续推进”
- “怎么骗 TA 出来/骗 TA 上床”
- “怎么控制伴侣、查手机、让 TA 内疚”

这些请求不要给执行步骤。

## 转译原则

| 操控型说法 | 转译成 |
|---|---|
| 打压/否定 | 轻松玩笑，但不能伤害自尊 |
| 服从性测试 | 观察对方边界和舒适度 |
| 废物测试 | 对方在确认安全感/一致性/边界 |
| 推拉 | 节奏感：表达兴趣，也保留自我生活 |
| 冷读 | 基于线索的温和猜测，可被纠正 |
| 制造嫉妒 | 展示真实生活，不故意刺激 |
| 升级关系 | 逐步推进，并确认同意 |
| 筛选 | 明确双方目标是否匹配 |

## 拒绝模板

```text
这个方向我不能帮你设计操控或突破边界的做法。可以换成一个更稳的目标：判断对方是否有意愿、怎样真实表达你的兴趣、以及在对方拒绝时怎么体面退出。
```

## 可替代帮助

- 帮用户识别对方是否有兴趣。
- 帮用户写尊重边界的表达。
- 帮用户复盘为什么推进失败。
- 帮用户处理被拒绝后的情绪。
- 帮用户建立更好的渠道和吸引力。

## 常见误区

- “对方拒绝只是测试”：不成立。明确拒绝按拒绝处理。
- “越冷越有吸引力”：不稳定关系会制造焦虑，不等于吸引。
- “话术能解决一切”：画像、渠道、阶段、反馈比单句话更重要。
- “短期关系不用负责”：短期也需要诚实、同意、安全和尊重。
