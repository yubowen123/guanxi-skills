# 画像采集与任务识别

## 先判断用户想要什么

把请求归入一个主任务：

| 任务 | 典型提问 | 优先补充信息 |
|---|---|---|
| 解决单身 | 找不到对象、没人聊、圈子小 | 城市、渠道、社交圈、形象、时间投入 |
| 具体对象 | 她/他对我有没有意思、怎么推进 | 认识渠道、互动记录、阶段、对方反馈 |
| 聊天回复 | 这句怎么回、冷场怎么办 | 原话、关系、目标、语气 |
| 暧昧推进 | 能不能升温、要不要表白 | 见面次数、主动性、身体/情绪边界 |
| 约会设计 | 第一次见面去哪、约后怎么跟进 | 城市、预算、关系阶段、对方偏好 |
| 相亲婚恋 | 相亲对象冷淡、条件匹配 | 婚恋目标、家庭因素、节奏、价值观 |
| 长期关系 | 吵架、冷战、信任、亲密下降 | 冲突史、共同生活、承诺状态 |
| 挽回/放下 | 分手后怎么聊、要不要复合 | 分手原因、对方态度、边界 |
| 成人约会 | 约炮、性暗示、亲密推进 | 成年确认、同意、安全、隐私、风险 |

## 沟通者画像

最小字段：

- 年龄、性别、性取向、城市。
- 职业/生活状态，时间自由度。
- 收入/资产大致区间，只问区间，不要求隐私细节。
- 外形和风格自评：干净度、穿搭、身材、照片质量。
- 恋爱经历：空窗、长期关系、相亲/社交软件经验。
- 社交圈：朋友介绍、工作圈、兴趣圈、线上平台。
- 关系目标：恋爱、婚恋、短期约会、性关系、复合、修复关系。
- 可投入资源：时间、精力、预算、情绪承受力。
- 底线边界：不接受什么关系、不想使用什么方式。

## 攻略目标画像

最小字段：

- 年龄、性别、性取向、城市、职业/生活状态。
- 关系状态：单身、暧昧中、有伴侣、已婚、不清楚。
- 认识渠道：相亲、朋友介绍、社交软件、工作、同学、线下搭讪、老关系。
- 当前阶段：刚认识、聊天中、见过面、暧昧、恋爱、长期关系、分手后。
- 对方反馈：回复速度、主动性、问题延展、邀约回应、身体/语言边界。
- 已知偏好：兴趣、作息、雷点、价值观、亲密边界。
- 风险点：权力差、职场关系、已有伴侣、异地、年龄差、隐私风险。

## 补问原则

- 先问最影响判断的问题，不要查户口。
- 用户着急要回复时，先给安全版回复，再说明需要哪些信息才能更准。
- 用户没有给性别/取向时，不默认“男追女”。
- 用户目标模糊时，先帮他/她把目标分成：修复、升温、邀约、筛选、结束。
