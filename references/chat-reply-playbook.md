# 聊天回复策略

## 回复前四问

1. 这句话是在提供信息、表达情绪、测试边界，还是结束话题？
2. 用户想要的结果是接住、升温、邀约、解释、道歉、筛选，还是结束？
3. 当前关系阶段允许多暧昧？
4. 对方是否给了继续聊的钩子？

## 回复原则

- 先接情绪，再接信息，再带方向。
- 能用真实生活细节，就不要用空泛套路。
- 不要连续追问；用“分享 + 轻问”替代盘问。
- 不要解释过度。解释越多，需求感越重。
- 调情必须轻、短、可退。对方没接就回到正常聊天。
- 邀约要具体，但给对方轻松拒绝空间。

## 常见场景

### 开场

目标：让对方容易接，不制造压力。

结构：

```text
共同点/场景 + 轻观察 + 小问题
```

示例方向：

- 社交软件：围绕资料里的具体点，不用“在吗”“你好漂亮”。
- 朋友介绍：先承接介绍关系，再给轻松话题。
- 相亲：简短说明自己是谁，再聊低压力生活话题。

### 冷场

先判断：是没话题、对方忙、兴趣不足，还是你连续追太紧。

可用动作：

- 停一停，等 1-2 天用真实生活小事重启。
- 从对方已知兴趣切入。
- 用低成本邀约替代硬聊。

避免：

- “怎么不回我？”
- “是不是我说错了？”
- 连续发多条解释。

### 对方说忙

好回复要接住、不追问、不阴阳怪气。

```text
行，你先忙。等你缓过来再说，我这两天也正好把手头事收一下。
```

如果已经聊得不错，可以轻带方向：

```text
那你先忙完，周末如果还有电量，我请你喝杯轻松的。
```

### 邀约

结构：

```text
具体理由 + 具体选项 + 低压力出口
```

示例：

```text
你上次说想吃东南亚菜，我刚好看到一家评价不错。周五晚或者周日下午你哪个更舒服？不赶的话我们可以去试试。
```

### 调情

只在 S2 末期以后使用。原则是可撤回、不过界、不物化。

轻度：

```text
你这个说法有点可爱，我先记一笔。
```

中度：

```text
你再这样讲，我会误会你是在给我制造见面的理由。
```

如果对方没接：

```text
哈哈我收一下，刚才有点皮。说正经的，你周末一般怎么安排？
```

### 道歉

结构：

```text
承认具体行为 + 说明理解到的影响 + 给对方空间 + 后续改变
```

不要用“如果你觉得被冒犯了”这种假道歉。

### 结束或筛选

当目标不一致时，清楚表达比拖着更好。

```text
我感觉我们对关系节奏的期待不太一样。我挺尊重你的选择，也不想互相消耗，就先到这里吧。
```

## 输出话术格式

给 1-3 条，标注风格：

- 稳妥版：低风险，适合不确定阶段。
- 升温版：适合已有好感。
- 收口版：适合邀约或结束。
